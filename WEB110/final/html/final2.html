Familiarize yourself with the CSS layout logic

If you don't understand how CSS is laid out you are not going to be able to effectively structure your site. The layout model is a grid going from top-to-bottom then left-to-right. Certain CSS display rules will alter this model.

Keep your styles scalable

Useful CSS3 Units
Unit Description Example Notes
pt Measures size in points. Points are 1/72 inches. font-size: 18pt; Great for things intended to be printed
px Measures size in pixels margin: 20px; Limit use; does not scale down well to mobile
em Measures size in em spaces. 2em equals double font size. font-size: 1.2em; Great for scalable text.
% Measure sizes as percentage of something else, usually parent element. width: 100%; Great for scaling objects relative to each other, i.e. headings and paragraphs.

Using px measurements was pretty popular when all of your users could be expected to be using a desktop monitor. With the proliferation of mobile devices, this assumption went out the window. Use em and percentage based measurements which scale better on alternative screens. px measurements can still be effective if you use small amounts (less than 10px is a good rule of thumb).

Consider using a CSS reset

Every browser has its own default style sheet. While these are typically minimal, slight quirks in each one can lead to a malformed site. You can use a CSS reset to level the field.

Consider designing for mobile first

It's a lot harder to get CSS right for mobile devices. If you design your site for mobile first, you have a lot more leeway in scaling it up to larger devices.

Utilize Developer Tools

Most modern browser come with a set of development tools that will help you design and troubleshoot your site. Typically these can be accessed by pressing the F12 key. They typically support the following features:

  • Show HTML and CSS syntax errors
  • Graphically display CSS box model attributes
  • Allow temporary modification of CSS and HTML for testing purposes
  • Show full derived CSS properties
  • Performance Benchmarking

This site was designed with the assistance of Microsoft Edge developer tools.


Utilize CSS minifiers

This doesn't really help you develop the CSS, but it is an important step in rolling out a website. A minifier will decrease file size allowing for increased bandwidth efficiency. Mobile users will love you for this.

Horizontal Centering — The Smart Way

To horizontally center elements, use the following:
selector {margin: 0 auto;}
setting the left and right margins to auto will keep the element horizontally centered with respect to its parent element.

Use the right DOCTYPE

Modern browsers are very smart and can make a lot of guesses about what you actually wanted your code to do, but if they have no idea what language you are speaking, they can't really make any guesses. The HTML5 DOCTYPE is as follows:
<!DOCTYPE html>

Avoid using browser specific CSS

Frequently in the past, and occasionally still today, it was necessary to use browser-specific CSS rules to create the desired style behavior. However these are largely obsolete and make CSS debugging troublesome across different browsers.

Validate your CSS

We all make mistakes. Unfortunately, browser developers are very forgiving of our mistakes and one can end up with an abomination of a site that does not appear at all as expected. Utilize the w3c's CSS validator to make sure your CSS is up to scratch.
