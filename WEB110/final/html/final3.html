Use concise naming

Use class and id naming that makes it clear what the rule is likely going to be applied to. #button is not a good identifier as it could be applied to any number of buttons on your page. #button-nav-home would be very clear that it is likely the home button on your navigation bar.

Do not mix style with naming

Avoid using style information in the name. The next person who works on the website is going to be very confused if he or she sees an id of #button-green and there are no green buttons on the site. Styles change over time so you shouldn't use style data in names.

Pick an organization convention (and stick with it)

CSS doesn't really care how you organize your code. For your own sanity however, consider implementing a consistent organization strategy. It may be alphabetical, or top-to-bottom. Most importantly, stick with this convention.

Here is an example convention:

  1. Body level selectors
  2. Header level selectors
  3. Nav level selectors
  4. Main level selectors
  5. Footer level selectors
  6. Class selectors
  7. ID selectors

Keep your code clean

During site development you may end up with duplicate selectors and rules. Try to keep these cleaned up as you go. It can be very confusing when you have a p selector defined in two different places and your site starts doing strange things.

Combine selectors with identical rulesets

If you have two or more selectors doing the exact same thing, combine the rule. This will save space and make your CSS more readable.

Use multiple classes

Already have two awesome rulesets that both do exactly what you need? Use both of them. You can declare as many classes as you like for a tag — just separate them with a space.

Utilize shorthand notation

Have you ever seen one of these monstrosities?
p {
margin-right: 5px;
margin-left: 6px;
margin-top: 10px;
margin-bottom: 10px;
}
Combine it to save space and improve readability:
p {margin: 10px 5px 10px 6px;}

Comment your Code

This might be the most underrated tip on this entire site. When you are actively coding CSS you know what you are doing. But if you ever have to shelve your project to work on another time, or if someone else has to take over your work for you, nobody is going to have any idea what is going on. I'm quite fond of a quote:

Always assume your successor is an axe murderer and knows where you live.

Avoid overusing <div>s

The div element is a very powerful and versatile for creating zones of content to style. However if a div is used for everything your style sheet will very quickly become overcluttered. Utilize the range of semantic tags added in HTML5.

Avoid permanently hiding elements with CSS

Some coders try to be clever with things such as h1 elements by setting massive negative margins or using display: none;. This can cause problems. Your CSS will be cluttered with rules that shouldn't really be there, and some search engines penalize sites for having non-rendering content.
